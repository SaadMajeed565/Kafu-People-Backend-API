# Bush.js Application

This is a starter application generated by the Bush.js CLI.

Use **bushjs `^0.1.8` or newer** (or a `file:` / workspace link to the framework repo) so middleware, console tooling, and structure config are available.

## Setup

1. Copy `.env.example` to `.env`
2. Run:
   ```bash
   npm install
   ```
3. With MongoDB running, apply schema files:
   ```bash
   node bush schema
   ```
4. Start development mode:
   ```bash
   npm run dev
   ```

## Project structure

- `app/` — application controllers and models
- `routes/` — route definitions
- `src/` — app entrypoint and startup modules
- `src/startup/` — auth, database, and middleware setup
- `config/` — configuration files

## Build

Compile TypeScript to JavaScript before production:

```bash
npm run build
```

## Start

Run the compiled app:

```bash
npm run start
```

## Framework console (Artisan-style)

From your project root:

```bash
node bush help
node bush make:model Product
node bush make:controller OrderController
node bush schema
node bush seed
```

Command help is grouped by category (General, Generators, Database, Monitoring, Backups).

### Path-aware database commands

Database commands support a `--path` override for custom layouts:

```bash
node bush make:schema create_users --path services/auth/database/schemas
node bush make:seeder initial_users --path services/auth/database/seeds
node bush schema --path services/auth/database/schemas
node bush seed --path services/auth/database/seeds
```

### Configurable project structure

For microservices/monorepos, set optional `STRUCTURE_*` vars in `.env` to control where generators and runtime schema/seed commands read/write:

- `STRUCTURE_CONTROLLERS_DIR`
- `STRUCTURE_MIDDLEWARE_DIR`
- `STRUCTURE_REQUESTS_DIR`
- `STRUCTURE_MODELS_DIR`
- `STRUCTURE_POLICIES_DIR`
- `STRUCTURE_ROUTES_DIR`
- `STRUCTURE_COMMANDS_DIR`
- `STRUCTURE_SCHEMAS_DIR`
- `STRUCTURE_SEEDERS_DIR`
